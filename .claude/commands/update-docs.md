# Update Documentation

Update project documentation based on current codebase.

## Usage

When asked to update documentation:
1. Review the current codebase
2. Identify what's missing or outdated
3. Update README.md with current setup instructions
4. Document new features or changes
5. Keep instructions simple and beginner-friendly

## Documentation Files

- `README.md` - Main project documentation
- Feature-specific `.md` files in root
- Code comments for complex logic

## Guidelines

- Use simple language (user is a beginner)
- Include step-by-step instructions
- Add code examples where helpful
- Keep it up to date with current code
