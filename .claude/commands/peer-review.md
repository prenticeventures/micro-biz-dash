# Peer Review

Perform a thorough code review as if reviewing a colleague's code.

## Usage

When asked for a peer review:
1. Examine the code structure and organization
2. Check for code quality and maintainability
3. Look for potential bugs or edge cases
4. Verify best practices are followed
5. Suggest improvements
6. Check for security issues
7. Ensure code is well-documented

## Review Criteria

- **Functionality**: Does it work correctly?
- **Code Quality**: Is it clean and readable?
- **Performance**: Are there optimization opportunities?
- **Security**: Any vulnerabilities?
- **Testing**: Is it testable?
- **Documentation**: Is it well-explained?

## Output

Provide constructive feedback with:
- What's working well
- Issues found
- Suggestions for improvement
- Priority level for each issue
