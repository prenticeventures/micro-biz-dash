# Code Review

Review code for issues, improvements, and best practices.

## Usage

When asked to review code:
1. Check for bugs and potential errors
2. Look for performance issues
3. Verify TypeScript types are correct
4. Check for security concerns
5. Suggest improvements
6. Ensure code follows project patterns

## Focus Areas

- Type safety
- React best practices
- iOS/Capacitor compatibility
- Database query efficiency
- Code clarity and comments
