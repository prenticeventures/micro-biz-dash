# Create Issue

Document a bug, feature request, or task as an issue.

## Usage

When asked to create an issue:
1. Clearly describe the problem or feature
2. Include steps to reproduce (for bugs)
3. Specify expected vs actual behavior
4. Add relevant context (files, error messages)
5. Suggest potential solutions if known

## Issue Format

**Title**: Clear, concise description

**Description**:
- What needs to be done
- Why it's needed
- How to reproduce (if bug)

**Context**:
- Relevant files
- Error messages
- Screenshots if applicable

**Acceptance Criteria**:
- What "done" looks like
- How to verify it's fixed
